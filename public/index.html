API Documentation

Authentication (api/auth)

The authentication API provides endpoints for user authentication and authorization.

  • POST /api/auth/register Register a new user.

    Creates a new user account.

  • POST /api/auth/login Log in with credentials and receive an access token.

    Logs in a user and returns an access token for authentication.

Users (api/users)

The users API allows you to manage user profiles and accounts.

  • GET /api/users/profile Get user profiles (admin only).

    Retrieves a list of user profiles.

  • GET /api/users/profile/:id Get user profile by ID.

    Retrieves the profile of a specific user by their ID.

  • PUT /api/users/profile/:id Update user profile by ID (user only).

    Allows a user to update their own profile information.

  • POST /api/users/profile/profilePhoto Upload user profile photo (logged-in users only).

    Enables logged-in users to upload a profile photo.

  • GET /api/users/count Get the count of users (admin only).

    Returns the total count of users in the system.

Posts (api/posts)

The posts API allows users to create, manage, and interact with posts.

  • POST /api/posts Create a new post.

    Enables users to create a new post with optional image attachment.

  • GET /api/posts Get all posts.

    Retrieves a list of all posts in the system.

  • GET /api/posts/count Get the count of posts.

    Returns the total count of posts available.

  • GET /api/posts/:id Get a single post by ID.

    Retrieves the details of a specific post by its ID.

  • PUT /api/posts/:id Update a post by ID.

    Allows users to update an existing post with optional image attachment.

  • DELETE /api/posts/:id Delete a post by ID.

    Enables users to delete a specific post by its ID.

  • PUT /api/posts/:id/like Toggle like on a post.

    Allows users to toggle their like on a specific post.

Comments (api/comments)

The comments API allows users to create, update, and delete comments on posts.

  • POST /api/comments Create a new comment.

    Enables users to create a new comment on a post.

  • GET /api/comments Get all comments (admin only).

    Retrieves a list of all comments in the system (admin access).

  • GET /api/comments/:id Get a comment by ID.

    Retrieves the details of a specific comment by its ID.

  • PUT /api/comments/:id Update a comment by ID.

    Allows users to update an existing comment.

  • DELETE /api/comments/:id Delete a comment by ID.

    Enables users to delete a specific comment by its ID.

Categories (api/categories)

The categories API deals with post categorization and management.

  • POST /api/categories Create a new category (admin only).

    Enables administrators to create a new category.

  • GET /api/categories Get all categories.

    Retrieves a list of all categories in the system.

  • DELETE /api/categories/:id Delete a category by ID (admin only).

    Allows administrators to delete a category by its ID.
